# 🧭 エンジニアリングマネージャー 学習ロードマップ

エンジニアリングマネージャー（EM）は、  
**技術 × 組織 × ビジネス** を横断的に扱い、チームを成功に導く役割を担います。

このリポジトリは、  
エンジニアからマネージャー、そして組織を率いるリーダーへと成長するための **学習指針と実践テンプレート** です。

---

## 🧩 スキル体系マップ

```
エンジニアリングマネージャー
├── 技術領域（Technical）
│ ├── 技術的基盤（アーキテクチャ理解・コード品質）
│ ├── プロセスとツール（CI/CD・SRE・開発プロセス）
│ └── 技術戦略と選定
│
├── 組織・マネジメント領域（People / Team）
│ ├── チームビルディング・心理的安全性
│ ├── 評価・育成・採用
│ └── プロジェクト運営・リスク管理
│
└── ビジネス・戦略領域（Business / Leadership）
├── 事業理解とKPI
├── 意思決定と優先順位付け
└── 組織戦略と文化形成
```
